# 舞台裏ローグライト・スコアアタック現行仕様

この文書は、`本番中 x 舞台裏` のローグライト・スコアアタック要素について、現行実装と今後の拡張方針をまとめた保守用仕様です。

既存のゲームシステム仕様、準備フェーズ UI ガイド、スコアアタック向け相性設計メモを前提に、反復プレイ、情報量整理、アイコン化、長期目標、同 seed 詰めの体験を統合する上位方針として扱います。

## 現行実装サマリー

実装済みの体験は、短い6ターン公演を何度も回し、`別seedで発見する`、`同seedで詰める`、`今日の巡り合わせを更新する` の3導線へ自然に戻す構造です。

| 領域 | 現行仕様 |
| --- | --- |
| 公演ビルド | 選択傾向から `heat`、`breath`、`control`、`closure` の型を集計し、型レベルと型ボーナスを終演評価へ反映する。 |
| 同seed再演 | 履歴内の同seed結果を参照し、準備/対応の前回情報、前回比、1ターン差し替えシミュレーションによる再演提案を表示する。 |
| 日替わり挑戦 | ローカル日付から `honban-daily-YYYY-MM-DD` seed を生成し、日替わり修飾子と自己ベストを表示する。 |
| 図鑑/称号 | 場面図鑑、代表未解放ヒント、称号を `localStorage` に保存し、タイトルと終演で進捗を見せる。 |
| 情報量整理 | 出来事/状態/対応/型/日替わりをアイコンと短いチップで提示し、詳細文は選択中パネル、結果画面、折りたたみの公演記録へ寄せる。 |
| 次の公演目標 | 終演後とタイトルで `同seed再演`、`日替わり`、`図鑑狙い`、`新seed` のうち1つを主CTAとして推薦する。 |
| 公演バッジ | 準備的中、名場面、低負荷、千秋楽場面化、型Lv.3など、その公演固有の達成を短いバッジで表示し、履歴フィルタへ使う。 |
| バランス確認 | `npm run balance:report -- --samples=48` で主要戦略のスコア、負荷、型、イベント分布を確認する。 |

主要な実装場所は以下です。

| ファイル | 役割 |
| --- | --- |
| `src/game/rogueliteProgress.ts` | 図鑑、称号、発見点、型集計、日替わり自己ベスト、再演前回比、次の公演目標の中核。 |
| `src/game/replayAnalysis.ts` | 同seed再演の1ターン差し替えシミュレーションと具体候補算出。 |
| `src/game/dailyRun.ts` | 日替わりseed、修飾子、イベント重み補正、負荷揺れ補正。 |
| `src/game/scoreRules.ts` | 対応採点、型レベルボーナス、日替わり負荷補正、文脈化された兆候。 |
| `src/game/performanceReport.ts` | 終演インサイト、公演バッジ、観客アンケート、劇評の生成。 |
| `src/game/actorLogic.ts` | 役者イベント重みと日替わり修飾子の適用。 |
| `src/app/usePerformanceHistory.ts` | 公演履歴、図鑑、日替わり自己ベストの保存副作用。 |
| `src/app/TitleScreen.tsx` | 日替わり挑戦、履歴再演、図鑑/称号/ヒントの入口。 |
| `src/app/ResultScreen.tsx` | 終演時の型、発見、前回比、日替わり自己ベスト、ポスター表示。 |
| `scripts/balance-report.mjs` | 依存追加なしのバランス集計レポート。 |

## 目標

- 1公演6ターンの短さを活かし、1回のプレイを軽く始められて、終演後に自然ともう1回始めたくなる構造にする。
- 読み筋の深みは維持しつつ、画面上の常時文字量を減らし、記号、灯り、メーター、アイコンで状況を読めるようにする。
- 同じ seed の再演で「どこを変えれば伸びるか」が明確に分かるスコアアタック体験にする。
- 別 seed の新規公演では、毎回違うビルド、巡り合わせ、収集要素が生まれるローグライト体験にする。
- 失敗も記録、図鑑、称号、次回課題へ変換し、終演結果がプレイヤーの成長につながるようにする。

## 体験の柱

### 1. 短いが濃い公演

現行の6ターン構成は維持する。長さを伸ばすのではなく、各ターンの判断密度と結果演出を上げる。

- 1プレイは5分から10分程度で終わる。
- 1ターンは `兆候を読む -> 準備 -> 本番対応 -> 結果` のまま維持する。
- 終演後は `別 seed`、`同 seed 再演`、`今日の巡り合わせ` へ即移動できる。

### 2. 今回だけの公演ビルド

1日目ソワレ後に決まる `公演の型` を、単なる補正ではなく今回のビルド軸に昇格する。

| 型 | ビルド名 | 育つ条件 | 主な報酬 |
| --- | --- | --- | --- |
| 熱量で押す公演 | 熱量型 | 拾う、名場面、評判上昇 | 評判倍率、客席熱量、派手な場面タイトル |
| 間で残す公演 | 余韻型 | 待つ、信頼、沈黙/退場の成功 | 千秋楽倍率、アンコール率、信頼ボーナス |
| 段取りで支える公演 | 精度型 | 整える、低負荷、流れ維持 | 事故軽減、安定点、ほころび抑制 |
| 崩れを閉じる公演 | 収束型 | 切る、高負荷局面の収束 | ほころび回収、下振れ軽減、低負荷終演 |

型は2日目以降に専用ゲージを持つ。ゲージは該当する判断で伸び、一定値に達すると型レベルが上がる。

```txt
型レベル 0: 未確立
型レベル 1: 公演の色が決まる
型レベル 2: 専用ボーナスが強化される
型レベル 3: 千秋楽フィニッシュ条件が開く
```

型レベルは終演タイトル、レビュー、図鑑、称号、スコア倍率に影響する。

### 3. seed を詰めるスコアアタック

同じ seed で再演した時は、単に最初からやり直すのではなく、前回との比較を常時支援する。

- タイトル履歴から同 seed を選ぶと `再演モード` として開始する。
- 各ターンに前回の準備、対応、結果ランクを薄く表示する。
- 結果画面では前回比を表示する。
- 終演画面では `伸びた点`、`落ちた点`、`最大改善ターン` を表示する。

比較表示は文字を増やしすぎず、ゴーストアイコンで示す。

| 表示対象 | 表示方法 |
| --- | --- |
| 前回の準備 | 準備カード右上に薄いゴーストアイコン |
| 前回の対応 | 対応カードに小さな履歴灯 |
| 前回より良い見立て | 緑の上向きマーカー |
| 前回より危険 | 赤い注意マーカー |
| 前回と同じ選択 | 連続再演マーカー |

### 4. 別 seed を回すローグライト

新規公演では、毎回の seed に `巡り合わせ` を付与する。

巡り合わせは、焦点役者やイベント抽選を壊さず、プレイ方針に軽い偏りを作る修飾子として扱う。

| 巡り合わせ | 効果 | 狙い |
| --- | --- | --- |
| 初日は荒れやすい | 1日目の本番の揺れ確率が少し高い | 守りの価値を上げる |
| 若手が乗りやすい | 若手焦点時、熱/前進/アドリブがやや出やすい | 拾う型を誘う |
| 主役が沈黙を抱える | 主役焦点時、沈黙/退場遅れがやや出やすい | 待つ型を誘う |
| 技巧派の軸が揺れる | 技巧派焦点時、位置/群像がやや出やすい | 整える型を誘う |
| 転換が重い | 道具/進行系ほころびが残りやすい | 切る/整えるの判断を増やす |
| 客席が熱い | 評判上昇が伸びるが、負荷も増えやすい | 攻めと代償を強める |

巡り合わせはタイトル画面で短いアイコン列として表示する。詳細は押した時だけ表示する。

## 反復プレイのループ

### 1公演内ループ

```txt
兆候を見る
-> 準備で読む
-> 本番で対応する
-> 結果で気持ちよく評価される
-> 次ターンの課題が1つ残る
```

次ターンの課題は必ず短くする。

良い例:

- `負荷が3。次は整えるか切ると安定。`
- `信頼が2。待つ成功で千秋楽が伸びる。`
- `光のほころびあり。拾うで回収可能。`

悪い例:

- 複数の理由を長文で並べる。
- 詳細スコア内訳を常時見せる。
- 用語説明を毎ターン繰り返す。

### 終演後ループ

```txt
終演ランクを見る
-> 場面図鑑/称号/ベスト更新を見る
-> 最大改善ターンを見る
-> 同 seed で詰めるか、新 seed へ行く
```

終演画面の主CTAは状況で変える。

| 状況 | 主CTA |
| --- | --- |
| 最大改善ターンが明確 | 同じ巡り合わせで詰める |
| 図鑑未解放が近い | 別の公演を開ける |
| 今日の巡り合わせ未プレイ | 今日の巡り合わせへ |
| ランク更新直後 | この seed をもう一度 |

## 収集・解放要素

### 場面図鑑

場面タイトルを収集対象にする。単なる一覧ではなく、達成条件と再現ヒントを持たせる。

| 項目 | 内容 |
| --- | --- |
| 場面タイトル | `拾われたアドリブ` など |
| 解放条件 | 役者、イベント、対応、結果ランク、ほころび回収 |
| 初回達成 seed | 達成した公演 seed |
| 最高ランク | その場面で出した最高結果 |
| 再現ヒント | 未達成時は伏せ字を混ぜる |

未達成時の表示例:

```txt
若手 / アドリブ / ？？？
ヒント: 予定外の一言を客席へ渡す。
```

### 称号

称号はプレイヤーの選択傾向や達成を記録する。

| 称号 | 条件 |
| --- | --- |
| 拾う手の演出助手 | 拾うで名場面を3回 |
| 間を信じた舞台監督 | 待つで千秋楽名場面 |
| 乱れを包む進行役 | 整えるでほころび回収 |
| 暗転の切れ味 | 切るで事故相当を小成功以上へ軽減 |
| 三日間を無傷で渡した | 最終負荷0または1 |
| 荒れた舞台を名作にした | ほころび2件以上からA以上 |

称号はゲーム内性能に影響しない。スコアアタックの公平性を守るため、収集はモチベーション報酬に留める。

### 演目ポスター

終演結果に応じて、簡易ポスターを生成する。

- 公演タイトル
- 型
- 最高場面
- 星評価
- 主要役者
- seed

画像生成ではなく、HTML/CSS上のカードとして表現する。後でスクリーンショット共有に使える。

## スコア設計

### 最終スコア

現行の総合点を拡張し、スコアアタックで分かりやすい内訳にする。

```txt
総合点 =
  評判 * 2
  + 段取り
  + 信頼
  - 裏方負荷 * 2
  + 型ボーナス
  + 図鑑初回ボーナス
  + ほころび回収ボーナス
  - 高負荷終演ペナルティ
```

注意:

- 図鑑初回ボーナスは通常ランキングには入れない。
- 同 seed スコアアタックでは純粋なプレイ結果だけを比較する。
- 初回達成の気持ちよさは別枠の `発見点` として扱う。

### スコア種類

| 種類 | 用途 | 図鑑/初回ボーナス |
| --- | --- | --- |
| 公演点 | 終演ランク、履歴比較 | 含めない |
| 発見点 | 初回解放、称号、図鑑 | 含める |
| seedベスト | 同 seed 再演比較 | 含めない |
| 今日の巡り合わせ点 | 日替わり挑戦 | 含めない |

### ランク

ランクは現行の方向を維持しつつ、次ランクまでの点数を常に出す。

```txt
C
B
A
S
S+
```

`S+` 以降はランキング的な数値詰めに入り、表示は `S+ / 上位更新` とする。ランクを増やしすぎない。

## 情報設計

### 原則

- 正確な判断に必要な数値は隠さない。
- 初見で読む必要がない説明文は常時表示しない。
- 常時表示は `状態`, `見立て`, `影響`, `次の一手` に絞る。
- 詳細は選択中パネル、ツールチップ、結果画面、図鑑へ逃がす。

### 画面別の情報量

| 画面 | 常時表示 | 詳細表示 |
| --- | --- | --- |
| タイトル | 今日の巡り合わせ、履歴ベスト、主CTA | 遊び方、図鑑、称号 |
| 準備 | 役者、状態、兆候アイコン、準備4択、カバー数 | 本番前メモ、備えどおり/外れたら |
| 対応 | 発生イベント、4択、結果レンジ、主要影響 | 相性4軸、パッシブ、ほころび、詳細内訳 |
| 結果 | ランク、場面タイトル、差分、次の判断 | 主な理由、代償、客席反応 |
| 終演 | ランク、次ランクまで、最大改善ターン、再演CTA | タイムライン、判断傾向、批評、図鑑更新 |

### アイコン化対象

| データ | 表示 |
| --- | --- |
| 準備 | 既存4アイコンを維持 |
| 対応 | 既存4アイコンを維持 |
| 兆候/出来事 | 8種類の専用アイコンを追加 |
| 役者状態 | 5種類の状態灯を追加 |
| 公演回 | 日/マチネ/ソワレをピクト化 |
| 型 | 4種類の型紋章を追加 |
| ほころび系統 | 光/音/進行/道具の4アイコン |
| 前回比 | 上昇/下降/同一/未比較の小アイコン |

文字を残すべき箇所:

- スコア名と数値。
- 場面タイトル。
- 結果ランク。
- CTA。
- アクセシビリティラベル。
- 初回チュートリアル。

## 準備フェーズ仕様

準備フェーズは `読む` 体験を保ちつつ、ファーストビューの文章量を減らす。

### 役者カード

焦点役者カードは以下へ整理する。

```txt
役者名
状態灯 + 状態名
兆候アイコン x3
```

役者タイプ説明は初回、または詳細展開時だけ表示する。

兆候は `高/中/低` の文字ではなく、灯りの強さで出す。正確なラベルはアイコン下またはツールチップに残す。

### 準備カード

準備カードの常時表示:

- アイコン
- 準備名
- 兆候カバー数
- 活きる対応アイコン
- リスク色

例:

```txt
注視
兆候 3/3
本番: 拾うアイコン
```

`見えている兆候に合う` などの文章は、選択中カードまたはメモへ移す。

### 本番前メモ

本番前メモは残す。ただし常時大きく読ませるのではなく、選択中準備の詳細確認として扱う。

- モバイルでは折りたたみ可能。
- 開いた時は既存のメモ体験を維持。
- CTA押下時は承認欄の演出を必ず見せる。

## 対応フェーズ仕様

対応フェーズはゲームの最重要判断画面とする。

### 発生イベント

発生イベントは画面の主役として維持する。

- イベント名を大きく表示。
- 役者名/状態は小さく添える。
- イベントアイコンと演出を追加する。

### 対応カード

4択カードの常時表示:

- 対応アイコン
- 対応名
- 結果レンジ
- 準備関係
- 主要影響2件
- 前回ゴースト

常時表示しない:

- 長い狙い説明。
- 上振れ/下振れ文章。
- 相性4軸の詳細。
- スコア内訳。

### 進行卓HUD

選択中対応の詳細を読む場所として強化する。

HUDは以下のモジュールを持つ。

| モジュール | 内容 |
| --- | --- |
| 見立て | 結果レンジ、ランク予測 |
| 相性盤 | 出来事、役者型、状態、公演回 |
| 影響 | 評判、流れ、信頼、負荷 |
| ほころび | 回収可否、系統 |
| 型 | 今の型ゲージへの影響 |
| 前回比 | 同 seed 再演時のみ |

モバイルでは対応カードの直下に `選択中HUD` を置き、CTAは追従バーにする。

## 結果フェーズ仕様

結果フェーズは報酬画面として強化する。

### 結果カード

常時表示:

- 結果ランク
- 場面タイトル
- 星/拍手/客席熱量
- スコア差分
- 次の判断メモ

詳細表示:

- 主な理由
- 代償
- 客席反応
- 準備回収テキスト
- 型ゲージ変化
- 図鑑更新

### 快感演出

| 結果 | 演出 |
| --- | --- |
| 名場面 | 照明が広がる、拍手、ポスター断片 |
| 場面化 | 舞台灯が安定する |
| 小さな成功 | 進行表にチェック |
| ほころび | 系統アイコンが次ターンへ残る |
| 事故 | 暗転、ただし次回課題を即提示 |

演出は短く、操作を待たせすぎない。`prefers-reduced-motion` を尊重する。

## 終演画面仕様

終演画面は `報酬`, `分析`, `次の一手` の3層に分ける。

### ファーストビュー

- 公演タイトル
- ランク
- 総合点
- 次ランクまで
- ベスト更新
- 図鑑/称号更新
- 主CTA

### 分析

- 最大改善ターン
- 準備ヒット率
- 名場面数
- ほころび/事故数
- 判断傾向
- 型レベル

### 記録

- タイムライン
- ベスト場面
- 批評
- 観客アンケート
- ポスター

## 現行データモデル

### 履歴と終演結果

`PerformanceResult` は公演点だけでなく、ローグライト表示に必要な派生情報を持つ。終演時に `src/game/rogueliteProgress.ts` で算出し、`src/app/usePerformanceHistory.ts` で履歴と図鑑へ保存する。

```ts
type PerformanceResult = {
  seed: string;
  finishedAt: string;
  sceneScore: number;
  flowScore: number;
  trustScore: number;
  backstageLoad: number;
  insight: {
    totalScore: number;
    rank: 'S+' | 'S' | 'A' | 'B' | 'C' | 'D';
    discoveryScore: number;
    buildStyle: BuildStyleSummary;
    unlockedAchievements: AchievementUnlock[];
    sceneCollectionCount: number;
  };
  logs: TurnLog[];
  highlights: TurnLog[];
};
```

`discoveryScore` は初回解放や称号などの発見報酬であり、同seed再演の純粋な比較には使わない。スコアアタックの公平性を守るため、今後も `totalScore` と `discoveryScore` は分離する。

### 図鑑保存

localStorage に別キーで保存する。

```txt
honban.collection.v1
```

```ts
type CollectionState = {
  scenes: Record<string, SceneCollectionEntry>;
  achievements: Record<string, AchievementEntry>;
};
```

日替わり自己ベストは図鑑とは別キーで保存する。

```txt
honban.daily.best.v1
```

履歴、図鑑、日替わり自己ベストはいずれもローカル保存であり、読み込み失敗時は空の状態として扱う。保存形式を変更する場合は、旧JSONの読み込み失敗でゲーム開始不能にならないようにする。

### 今日の巡り合わせ

日付と固定ソルトから seed を生成する。

```txt
dailySeed = honban-daily-YYYY-MM-DD
```

タイムゾーンは端末ローカルでよい。オンラインランキングを導入するまではサーバー時刻を使わない。

現行の日替わり修飾子は `src/game/dailyRun.ts` の `DailyModifierId` で管理する。追加時は以下を同時に更新する。

- `DAILY_MODIFIERS`
- `modifierForSeed`
- `applyDailyEventWeights`
- `dailyVolatilityBonus`
- タイトル画面の短縮表示
- `tests/roguelite-progress.test.ts`

## 実装フェーズ

### Phase 1: 情報量整理

状態: 実装済み。

- 兆候/出来事/状態/型/ほころびのアイコン体系を追加済み。
- 準備カードと対応カードは短いチップ、アイコン、差分中心に整理済み。
- 対応カードは `結果レンジ + 準備関係 + 主要影響 + 前回比` を常時表示する。
- 詳細は進行卓、結果プレビュー、終演分析へ逃がす。

### Phase 2: 再演スコアアタック

状態: 実装済み。

- 同seedの履歴を参照し、履歴から同じ巡り合わせを再演できる。
- 準備カードに前回マーカーを表示する。
- 対応見立てと終演画面で前回比を表示する。カード上の重複は抑え、選択中HUDへ寄せる。
- 最大改善ターンを選び、そのターンだけ準備/対応を差し替えた場合の終演スコアを再シミュレーションする。
- 具体候補がある場合は `締め -> 整える / 総合+6` のように、次の公演目標として表示する。
- 同seed再演中は準備承認演出を短縮し、テンポよく詰められるようにする。

### Phase 3: 型ビルド強化

状態: 実装済み。演出強化余地あり。

- 型ゲージ、型レベル、型専用ボーナスを追加済み。
- 結果画面と終演画面で型を表示する。
- 型レベル2以上は採点に加点する。
- 終演タイトル/レビュー/称号に型達成を反映する。今後はターン中の型ゲージ変化演出をさらに強める。

### Phase 4: 図鑑と称号

状態: 実装済み。拡張継続。

- 場面図鑑、代表未解放ヒント、称号を追加済み。
- タイトル画面から図鑑へ入り、進捗、最新解放、ヒントを見られる。
- 終演時に新規場面、称号、発見点を表示する。
- 終演時に公演バッジを表示し、タイトルの履歴をバッジで絞り込める。
- 新しい役者イベントや場面タイトルを追加した場合は、図鑑条件とヒントも同時に増やす。

### Phase 5: 今日の巡り合わせ

状態: 実装済み。

- daily seed を追加済み。
- 巡り合わせ修飾子をイベント重みと負荷揺れへ反映する。
- タイトル画面に今日の挑戦、特徴、自己ベストを表示する。
- 今日の自己ベストを `honban.daily.best.v1` に保存する。

## 保守チェックリスト

ローグライト/スコアアタック関連を変更した場合は、変更範囲に応じて以下を確認する。

```bash
npm run test:logic
npm run balance:report -- --samples=48
npm run check:ui
```

- 採点、イベント重み、日替わり補正、型ボーナスを変えたら `balance:report` で主要戦略の平均点、負荷、型分布が極端に偏っていないかを見る。
- UI文言、カード密度、図鑑/履歴表示を変えたら `check:ui` を使う。通常はPNGを作らない。
- 保存キーや保存形式を変える場合は、旧データ読み込み失敗時のフォールバックを維持し、ゲーム開始不能にしない。
- 永続解放は見た目、記録、ヒントに留める。プレイヤー性能を上げる永続強化はスコアアタックの公平性を壊すため追加しない。

## 非目標

- 課金、スタミナ、ログインボーナスのような外発的な継続圧は入れない。
- プレイヤー性能を上げる永続強化は入れない。スコアアタックの公平性を守る。
- UIから数値を完全に消さない。必要なスコア、差分、負荷は正確に表示する。
- 1公演のターン数を安易に増やさない。

## 成功基準

- 初回プレイヤーが、準備と対応の意味を文章説明なしでもおおよそ読める。
- 2回目以降のプレイヤーが、終演後に次の挑戦理由を1つ持てる。
- 同 seed 再演で、前回と違う判断を試したくなる。
- 別 seed 開始で、今回の公演ビルドを考えたくなる。
- 結果画面を見た時に、成功も失敗も次の行動へつながる。
