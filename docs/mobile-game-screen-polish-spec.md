# スマホ主導ゲーム画面改善仕様

この文書は、現行実装を前提に、スマホ縦画面での `一瞬で状況を読める`、`結果が気持ちいい`、`次の挑戦目標が勝手に立つ` 体験を強めるための画面改善仕様です。

## 前提

- 1公演6ターン、`準備 -> 対応 -> 場面` の進行は維持する。
- 既存の舞台裏らしい質感、メモ紙、進行卓、結果票の方向性は維持する。
- 画面数は増やさず、各画面の表層情報と詳細情報を分ける。
- 正確な数値は消さない。ただし主要画面では、数値より先にアイコン、灯り、レール、スタンプで読めるようにする。
- 初回説明を削りすぎない。初見が読むべき説明は短く、細かい理由は選択中詳細や記録へ寄せる。
- プレイ中UIは最善手を出さない。出すのは判断材料、危険、可能性、選択の性格までにする。
- 機能を増やす前に、既存表示の統合、削除、詳細層への移動を優先する。
- タイトル画面は初回プレイヤーをすぐ開始させる。目標や条件を開始前に読ませすぎない。
- 次の挑戦提示は必須機能ではない。うまく機能しない場合は削り、終演時の点差と不足表示を優先する。

## 現行実装の読み取り

| 領域 | 現行実装 | 改善方針 |
| --- | --- | --- |
| アプリ進行 | `src/app/App.tsx` が `title/prep/response/result/finished` を切り替える | 進行構造は維持し、各フェーズ内の読む順を整理する |
| ヘッダー | `GameHeader` がターン、日、公演の色、日替わりを表示する | 上部はこれ以上増やさず、下の主画面で判断軸を強める |
| スコア | `ScoreBar` が評判、段取り、信頼、裏方負荷、ほころびを表示する | 負荷とほころびを警戒サインとして強め、3スコアは小型化する |
| 役者表示 | `ActorStage` が焦点役者、状態、兆候、控え、発生イベントを表示する | 準備時は兆候3灯を主役化し、対応時は発生イベントをさらに主役化する |
| 準備 | `PrepPanel` が4択、カバー数、活きる対応、本番前メモ、承認を表示する | カード表面は記号中心、メモ紙は詳細確認と承認演出へ寄せる |
| 対応 | `ResponsePanel` が4択、結果レンジ、主要影響、送出バー、進行卓を表示する | 4択カードは勝ち筋を瞬読化し、進行卓は選択中の詳細に限定する |
| 結果 | `ResultPreviewCard` が場面タイトル、ランク、準備、理由、差分、次メモを表示する | ファーストビューを報酬画面にし、理由や代償は下層に送る |
| 終演 | `ResultScreen` がランク、点数、次公演推薦、指標、記録を表示する | ランク直下に点差と足りない部分を置き、再演導線は主張しすぎない |
| 収集/再演 | `rogueliteProgress` が次挑戦、図鑑、称号、日替わり、前回比を算出する | ロジックは残すが、タイトル初回や終演主導線で答えに見せない |
| アイコン | `Icon` が準備、対応、出来事、スコアの独自アイコンを持つ | 既存アイコンを中心に、状態灯、型紋章、ほころび系統の表現を拡張する |

## 体験原則

### 1. 主要画面は3秒で読む

各フェーズのファーストビューは、次の問いに3秒で答えられることを成功条件にする。

| フェーズ | 3秒で答える問い |
| --- | --- |
| 準備 | 誰が揺れていて、どの兆候が濃く、各準備が何に備えるか |
| 対応 | 何が起きて、各対応が何を伸ばし、何を危険にするか |
| 場面 | どう仕上がり、何が伸び、次に何を直すか |
| 終演 | 何ランクで、あと何が足りず、次に何を試せそうか |

### 2. 数値は正確さ、灯りは優先度

- 評判、段取り、信頼、負荷の数値は残す。
- 初見で読ませたい優先度は、灯り、色、アイコン、レールで先に示す。
- 数値を読む前に、`危ない`、`伸びそう`、`準備が効いた`、`次はここ` が見える状態にする。

### 3. 舞台の雰囲気は詳細層で濃くする

- 主要カードの文章を短くしても、メモ紙、進行卓、結果票の本文は残す。
- 説明文は常時全面に置かず、選択中カード、展開パネル、結果詳細、終演記録へ寄せる。
- 説明を消すのではなく、読むタイミングを分ける。

### 4. 正解ではなく仮説を出す

- プレイ中は `おすすめ`、`最善`、`最高得点`、`総合+N` のような正解提示をしない。
- プレイ中に表示するのは `備えの範囲`、`結果の幅`、`主な影響`、`危険の有無`、`選択の性格` に留める。
- 終演後は再演の仮説を出してよい。ただし断定ではなく、`試せる筋`、`詰めどころ`、`改善余地` として扱う。
- 同seed再演の具体候補はスコアアタック報酬として残すが、タイトル/CTAでは答えに見えない言い方にする。

## 表示統廃合方針

新しい部品を足す前に、既存表示を以下のように統合する。

| 現行表示 | 統廃合後 | 理由 |
| --- | --- | --- |
| 評判/段取り/信頼の大きめチップ + 負荷 console | 小型3スコア + 大きめ負荷 console | 常時判断では負荷とほころびが優先されるため |
| 役者タイプ説明 + 状態説明 + 兆候リスト | 焦点役者モニター + 状態1行 + 兆候3灯 | 準備時の主判断を兆候に寄せるため |
| 準備カードの相性文 + カバー数 + 活きる対応文 | 準備カードのカバー灯 + 対応アイコン + 短い狙い | 文章の重複を減らし、答え表示に見せないため |
| 準備カード詳細 + 本番前メモ | カードは要約、メモ紙は詳細と承認 | メモ紙の舞台裏感を残しつつ表層を軽くするため |
| 対応カードの戦術文 + 準備関係 + 主要影響 | 対応カードは性格/幅/影響、詳細は進行卓 | 4択比較と選択中詳細を分けるため |
| 結果の決め手/代償/申し送り/客席/理由が同列 | 結果上部はランク/場面/差分/次メモ、理由は下層 | 報酬と学びを先に読ませるため |
| タイトルの開始ボタン + 推薦パネル | 初回は開始CTA中心、履歴ありは控えめな再演導線 | 初見に不明な条件を読ませないため |

削除または詳細層へ移す候補:

- 役者タイプ説明の常時表示。
- 準備カード上の長い相性文。
- 対応カード上の長い狙い説明。
- 対応カード上の前回ゴースト表示。
- 結果ファーストビュー内の主な理由リスト。
- 終演ファーストビュー内の詳細レビュー2文目以降。

## 共通レイアウト仕様

### 上部HUD

対象: `GameHeader`, `ScoreBar`, `phase-strip`

- ヘッダーは現行の `1/6`, `1日目 マチネ`, `公演の色` を維持する。
- 上部HUDの `信頼` は座組全体の信頼として扱う。役者個別信頼とは混ぜない。
- スコアは以下の優先度にする。
  1. 裏方負荷5灯
  2. ほころび/乱れそう警告
  3. 評判/段取り/座組信頼の小型数値
- 負荷が3以上なら、負荷行だけでなくスコアレール外枠にも警戒トーンを入れる。
- フェーズ表示はモバイルでは現行どおり非アクティブの文字を抑え、アクティブフェーズだけ読ませる。

### 主要CTA

- 準備、対応、結果、終演の主CTAは常に親指で押しやすい下部寄りに置く。
- `この準備で本番へ`、`このキューを出す`、`この結果で次の回へ` は文言を維持する。
- 結果では、主CTAの直前に必ず `次の判断メモ` が見えるようにする。
- 終演では、主CTAの近くに `あとN点`、`足りない部分`、再演/別公演の導線を置く。次の挑戦カードは必須にしない。

## 準備画面仕様

対象: `ActorStage`, `PrepPanel`

### ファーストビュー構成

準備画面の読む順は次に固定する。

1. 焦点役者モニター
2. 兆候3灯
3. 準備4択
4. 準備CTA
5. 本番前メモ

### 焦点役者モニター

現行の焦点役者カードを、モバイルでは次の情報に絞る。

```txt
焦点役者
若手
状態灯 高揚
個別信頼/パッシブ
兆候3灯: 前へ出る / アドリブ / 熱が乗る
```

- 役者タイプ説明は詳細表示へ寄せる。
- 状態説明は役者カード上では出しすぎない。役者カードでは状態名と状態灯を主にする。
- 個別信頼や `阿吽の呼吸` などのパッシブは役者に関する情報なので、焦点役者カード内に置く。
- シルエットは残すが、判断情報より大きくしすぎない。
- 控え役者は `次に来そう` だけを小さく表示し、通常の控え説明は抑える。

### 兆候3灯

- `高/中/低` 文字はアクセシビリティと詳細には残す。
- 表面では灯りの強さ、アイコンの明度、並び順で強度を表す。
- 準備カード側にも同じ3灯を小さく表示し、どの準備がどの兆候を覆うかをチェック/消灯で示す。
- 3灯は `最善` ではなく `備えの範囲` として扱う。3灯点灯していても、負荷、型、外れた時の受け方により別準備を選ぶ余地を残す。

### 準備カード

カード表面の常時表示は以下に限定する。

```txt
アイコン + 準備名
兆候カバー 3灯
活きやすい対応アイコン
備える揺れカテゴリ
前回マーカー
```

例:

```txt
注視
● ● ●
拾うアイコン
熱の揺れ
```

- `熱を拾う` のような短文は、対応の最善手を示す言葉に見えやすい。カード表面では `熱の揺れ`、`間の揺れ`、`進行の乱れ`、`転換の崩れ` のように、準備が備える対象カテゴリとして表す。
- `見えている兆候に合う` などの文章は、選択中状態またはメモ紙へ寄せる。
- `本番で拾う` は行動固定に見えやすいので、カード表面では対応アイコンだけを基本にする。
- 危険/別筋は赤文字説明ではなく、カード外枠とカバー灯の欠けで先に伝える。
- カード上に `おすすめ` や `最適` は出さない。

### 準備CTA

- `この準備で本番へ` は準備4択の直下に置く。
- CTAは選択中の準備を本番前メモへ承認する操作として扱う。
- 本番前メモの中に押せる承認操作は置かない。
- CTAの下に本番前メモが続くことで、軽く遊ぶプレイヤーはカードとCTAだけで進み、読み込むプレイヤーはそのままメモを読める。

### 本番前メモ

- 現行のメモ紙は残す。
- 開閉式にはしない。毎ターン詳細確認するプレイヤーに余計な手間を増やさない。
- モバイルでは密度を整理し、選択中準備の詳細確認として扱う。
- 状態の意味説明は本番前メモに置く。例: `高揚: 熱が乗りやすく、拾うと伸びやすい。`
- 承認CTA押下時は、現行どおりメモ紙上の承認欄が `未承認 -> 承認済` へ変わる。
- 承認演出は短く、再演モードでは現行の短縮テンポを維持する。

## 対応画面仕様

対象: `ActorStage` のイベント表示、`ResponsePanel`, `ResponseChoiceCard`, `ResponseConsole`

### 発生イベント

- 対応画面の主役はイベント名にする。
- `本番で起きた`、イベント名、イベントアイコン、役者/状態の順に読む。
- イベント説明は3行まで。詳細は進行卓側に寄せる。
- 可能ならイベント種別ごとに光の動きや背景ラインを変える。

### 対応カード

カード表面の常時表示は以下に限定する。

```txt
対応アイコン + 対応名
手触りラベル
結果レンジレール
準備関係
主要影響2件
```

手触りラベル:

| 対応 | 表示 | 役割 |
| --- | --- | --- |
| 拾う | 熱を渡す | 評判を伸ばしやすいが、負荷が残ることもある |
| 整える | 呼吸を戻す | 流れを戻しやすいが、評判は伸び切らないこともある |
| 待つ | 間を残す | 信頼や余韻を残しやすいが、テンポには注意が要る |
| 切る | 次へ送る | 崩れを閉じやすいが、見せ場を捨てることもある |

- `攻め手/安定手` のような抽象バッジは増やさない。
- 結果レンジレールは現行を維持し、現在見立ての位置をより強く光らせる。
- 主要影響は `評/流/信/負` の文字ではなく、既存スコアアイコンと増減記号で表示する。正式名はariaと詳細で補う。
- 危険警告はカード下部に強く表示する。ただし通常時に事故文言を出しすぎない。
- カード間の優劣順位は表示しない。比較できる材料だけを並べる。
- `名場面狙い` などの結果見立ては幅として扱い、確定予告に見せない。
- 前回ゴーストは対応カード表面には出さない。同seed再演中に必要な場合だけ、進行卓または結果後の比較へ寄せる。

### 進行卓

- 選択中対応の詳細を読む場所として維持する。
- 相性盤、影響ゲージ、ビルド、ほころび、前回比は進行卓側に置く。
- 判定点、次ランクまでの差、見立ての補足説明は進行卓へ常時表示しない。検算は結果画面の折りたたみ、仕様確認は遊び方タブへ分ける。
- モバイルでは進行卓の上部に、選択中対応の要約を1行で固定表示する。
- 進行卓内にCTAは入れない。現行どおり送出バーで分離する。
- 同seed再演の前回情報は、通常プレイの4択比較を邪魔しない範囲で進行卓に表示する。

## 場面結果画面仕様

対象: `ResultPreviewCard`

### ファーストビュー構成

結果画面の上部は報酬画面として再構成する。

1. 結果ランク
2. 場面タイトル
3. 星/拍手/客席熱量
4. ルート: 役者 -> 出来事 -> 対応
5. 差分4つ
6. 次の判断メモ

理由、代償、申し送り、客席反応、準備回収詳細、公演の色、主な理由、判定内訳はその下に置く。判定内訳は折りたたみにし、通常の報酬感を邪魔しない。

次の判断メモは、単一の正解ではなく選択余地を残す。

良い例:

```txt
負荷が3。次は整えるか切ると安定。
信頼が2。待つ成功で千秋楽が伸びる。
光のほころびあり。拾うで回収を狙える。
```

避ける例:

```txt
次は締めを選ぶべき。
次は整えるが最善。
このseedは3日目に拾うと+6。
```

### 報酬演出

| 条件 | 表示 |
| --- | --- |
| 名場面 | 照明が広がる、星/拍手を強く出す |
| 場面化 | 舞台灯が安定する |
| 小さな成功 | 進行表チェックを出す |
| ほころび | 残った系統アイコンを次ターンへ渡す |
| 事故 | 暗転気味。ただし次の判断メモをすぐ見せる |
| 準備が活きた | メモ紙由来のスタンプを大きく出す |
| 新場面候補 | 図鑑スタンプを出す |
| ほころび回収 | ほころび系統アイコンを回収演出する |

### 差分表示

- 現行の `Delta` は維持する。
- 表面では `評判 +3` より先に `評判が伸びた` を読ませる。
- 負荷はプラスが悪化、マイナスが改善であることが一目で分かる色にする。

## 終演画面仕様

対象: `ResultScreen`, `PerformancePoster`

### ファーストビュー

終演画面の上部は次の順にする。

1. 公演タイトル
2. ランク/総合点/次ランクまで
3. 足りない部分の要約
4. 今回の公演バッジ
5. 主要指標

レビュー本文は残すが、モバイルでは初期表示を1文から2文に抑える。

### 不足表示と再演導線

次の挑戦カードは必須ではない。終演画面では、まずスコアアタックとして必要な `自分の点`、`次ランクまでの差`、`足りない部分` を見せる。

表示内容:

```txt
総合評価点 47
あと5点で S+
不足: 準備あと1回 / 負荷を2以下へ
```

- 推薦ロジックは履歴ありタイトルや詳細記録の補助に留め、終演ファーストビューでは点差と不足を優先する。
- 推薦カードがうまく機能しない場合は、無理に出さない。
- 主導線は `同じ巡り合わせでやり直す` と `別の公演を開ける` で十分に成立する。
- 具体的な差し替え候補や点数差分は、折りたたみの再演メモや詳細記録へ置く。
- 文言は `不足`、`あとN点`、`狙える` を使い、`これが正解` に見える表現を避ける。

### 指標

- `準備 5/6`, `名場面 1`, `場面以上 3`, `ほころび 1` は残す。
- ただしファーストビューでは `あと5点でS+` や `準備あと1回` のような課題文を指標より上に置く。
- 詳細の観客アンケート、劇評、判断傾向、タイムラインは現行どおり折りたたみに置く。

## タイトル画面仕様

対象: `TitleScreen`

### ホーム

- 最初にタイトル、雰囲気、開始CTAを表示する。
- 初回は `新しい公演を始める` を最も目立たせる。
- 初回に目標、条件、次の挑戦カードを読ませない。
- 履歴がある場合だけ、開始CTAの下に控えめな再演導線を置く。
- 新規開始、再演、日替わりを横並びの同格ボタンにしない。
- 記録、図鑑、遊び方は現行タブを維持する。

### 初回導線

- 初回は遊び方を長く読ませず、すぐ始められる状態にする。

```txt
[新しい公演を始める]
```

- 遊び方タブは現行の3ステップを維持し、必要な人だけ読める場所にする。
- 判定仕様は遊び方タブ内の折りたたみとして置く。ランク閾値、準備上限、連続使用、負荷、ほころび、公演の色を短く確認できるようにする。
- 必要なら将来的に初回プレイ中だけ、小さなツールチップで `準備を選ぶ`、`キューを出す` を補助する。ただし今回の仕様では未実装とする。

## コピー仕様

### カード表面で使う短い言葉

| 用途 | 文言例 |
| --- | --- |
| 準備の対象 | 熱の揺れ / 間の揺れ / 進行の乱れ / 転換の崩れ |
| 準備関係 | 活きる / 備え外 / 合わない |
| 兆候カバー | 3/3 ではなく3灯で表示。ariaには `3件に備え` のように短くする |
| 結果見立て | 名場面狙い / 場面化狙い / 小さく成功 / 崩れ抑え / 事故注意 |
| 影響 | 評判/段取り/信頼/負荷のアイコン + 増減記号 |
| 終演不足 | あと5点でS+ / 準備あと1回 / 負荷を2以下へ |

避ける言葉:

- `おすすめ`
- `最善`
- `正解`
- `最高効率`
- `総合+N` をプレイ中またはタイトル画面に出すこと
- `選ぶべき`

### 詳細層に残す言葉

- 役者タイプ説明
- 状態の意味
- 備えどおり/外れた場合の読み筋
- 相性盤4軸
- 決め手、代償、客席反応
- 公演レビュー、劇評、タイムライン

## アイコン/記号仕様

既存の `Icon` を優先して使う。追加する場合は同じ線幅、角丸、`currentColor` の方針に揃える。

追加候補:

| 対象 | 表現 |
| --- | --- |
| 状態 | 高揚/沈思/不安/没入/疲労の状態灯 |
| 公演の色 | 熱量/余韻/精度/収束の型紋章 |
| ほころび | 光/音/進行/道具の系統アイコン |
| 前回比 | 上昇/同じ/下降の小マーカー |
| 新規解放 | 図鑑スタンプ |

## モーション仕様

- モーションは判断を待たせない。
- 重要な瞬間だけ短く入れる。
- `prefers-reduced-motion` では点灯/色変化だけにする。

| 瞬間 | 推奨演出 |
| --- | --- |
| 準備選択 | 準備カードのカバー灯が点く |
| 準備承認 | メモ紙の承認欄に押印 |
| イベント発生 | イベントカードに照明スイープ |
| 対応選択 | 結果レンジの現在位置が点灯 |
| キュー送出 | 送出バー左右ランプが流れる |
| 名場面 | 結果タイトル周辺に短い照明拡張 |
| ほころび回収 | 系統アイコンが結果票へ吸い込まれる |
| ベスト更新 | 終演カードにスタンプ |

### Step 4: 終演不足表示とタイトル導線整理

対象:

- `src/app/TitleScreen.tsx`
- `src/app/ResultScreen.tsx`
- `src/game/rogueliteProgress.ts`
- `src/content/ja/appCopy.ts`
- `src/styles/global.css`

内容:

- タイトルホームは開始CTAを主役にし、初回は目標カードを出さない。
- 履歴がある場合だけ、控えめな再演導線を開始CTAの下に置く。
- 終演ではランク直下に `あとN点` と `足りない部分` を置く。
- 次の挑戦カード/パネルは必須ではない。使う場合も終演補助または履歴ありホームの控えめな導線に留め、タイトル主導線にはしない。
- 具体点数差分は詳細/再演メモへ移す。

確認:

```bash
npm run check:ui
```
