# スマホ主導ゲーム画面改善仕様

この文書は、現行実装を前提に、スマホ縦画面での `一瞬で状況を読める`、`結果が気持ちいい`、`次の挑戦目標が勝手に立つ` 体験を強めるための画面改善仕様です。

## 前提

- 1公演6ターン、`準備 -> 対応 -> 場面` の進行は維持する。
- 既存の舞台裏らしい質感、メモ紙、進行卓、結果票の方向性は維持する。
- 画面数は増やさず、各画面の表層情報と詳細情報を分ける。
- 正確な数値は消さない。ただし主要画面では、数値より先にアイコン、灯り、レール、スタンプで読めるようにする。
- 初回説明を削りすぎない。初見が読むべき説明は短く、細かい理由は選択中詳細や記録へ寄せる。

## 現行実装の読み取り

| 領域 | 現行実装 | 改善方針 |
| --- | --- | --- |
| アプリ進行 | `src/app/App.tsx` が `title/prep/response/result/finished` を切り替える | 進行構造は維持し、各フェーズ内の読む順を整理する |
| ヘッダー | `GameHeader` がターン、日、公演の色、日替わりを表示する | 上部はこれ以上増やさず、下の主画面で判断軸を強める |
| スコア | `ScoreBar` が評判、段取り、信頼、裏方負荷、ほころびを表示する | 負荷とほころびを警戒サインとして強め、3スコアは小型化する |
| 役者表示 | `ActorStage` が焦点役者、状態、兆候、控え、発生イベントを表示する | 準備時は兆候3灯を主役化し、対応時は発生イベントをさらに主役化する |
| 準備 | `PrepPanel` が4択、カバー数、活きる対応、本番前メモ、承認を表示する | カード表面は記号中心、メモ紙は詳細確認と承認演出へ寄せる |
| 対応 | `ResponsePanel` が4択、結果レンジ、主要影響、送出バー、進行卓を表示する | 4択カードは勝ち筋を瞬読化し、進行卓は選択中の詳細に限定する |
| 結果 | `ResultPreviewCard` が場面タイトル、ランク、準備、理由、差分、次メモを表示する | ファーストビューを報酬画面にし、理由や代償は下層に送る |
| 終演 | `ResultScreen` がランク、点数、次公演推薦、指標、記録を表示する | 次の挑戦カードをタイトル/終演で同じ見た目に統一する |
| 収集/再演 | `rogueliteProgress` が次挑戦、図鑑、称号、日替わり、前回比を算出する | ロジックは活かし、推薦が埋もれないUIにする |
| アイコン | `Icon` が準備、対応、出来事、スコアの独自アイコンを持つ | 既存アイコンを中心に、状態灯、型紋章、ほころび系統の表現を拡張する |

## 体験原則

### 1. 主要画面は3秒で読む

各フェーズのファーストビューは、次の問いに3秒で答えられることを成功条件にする。

| フェーズ | 3秒で答える問い |
| --- | --- |
| 準備 | 誰が揺れていて、どの兆候が濃く、どの準備が一番噛みそうか |
| 対応 | 何が起きて、どの対応が伸び筋/守り筋/危険筋か |
| 場面 | どう仕上がり、何が伸び、次に何を直すか |
| 終演 | 何ランクで、あと何が足りず、次は何を押すべきか |

### 2. 数値は正確さ、灯りは優先度

- 評判、段取り、信頼、負荷の数値は残す。
- 初見で読ませたい優先度は、灯り、色、アイコン、レールで先に示す。
- 数値を読む前に、`危ない`、`伸びそう`、`準備が効いた`、`次はここ` が見える状態にする。

### 3. 舞台の雰囲気は詳細層で濃くする

- 主要カードの文章を短くしても、メモ紙、進行卓、結果票の本文は残す。
- 説明文は常時全面に置かず、選択中カード、展開パネル、結果詳細、終演記録へ寄せる。
- 説明を消すのではなく、読むタイミングを分ける。

## 共通レイアウト仕様

### 上部HUD

対象: `GameHeader`, `ScoreBar`, `phase-strip`

- ヘッダーは現行の `1/6`, `1日目 マチネ`, `公演の色` を維持する。
- スコアは以下の優先度にする。
  1. 裏方負荷5灯
  2. ほころび/乱れそう警告
  3. 評判/段取り/信頼の小型数値
- 負荷が3以上なら、負荷行だけでなくスコアレール外枠にも警戒トーンを入れる。
- フェーズ表示はモバイルでは現行どおり非アクティブの文字を抑え、アクティブフェーズだけ読ませる。

### 主要CTA

- 準備、対応、結果、終演の主CTAは常に親指で押しやすい下部寄りに置く。
- `この準備で本番へ`、`このキューを出す`、`この結果で次の回へ` は文言を維持する。
- ただし結果/終演では、主CTAの直前に必ず `次の判断メモ` または `次の挑戦カード` が見えるようにする。

## 準備画面仕様

対象: `ActorStage`, `PrepPanel`

### ファーストビュー構成

準備画面の読む順は次に固定する。

1. 焦点役者モニター
2. 兆候3灯
3. 準備4択
4. 本番前メモ
5. 承認CTA

### 焦点役者モニター

現行の焦点役者カードを、モバイルでは次の情報に絞る。

```txt
焦点役者
若手
状態灯 高揚
兆候3灯: 前へ出る / アドリブ / 熱が乗る
```

- 役者タイプ説明は詳細表示へ寄せる。
- 状態説明は `拾うと伸びやすい` 程度の1行にする。
- シルエットは残すが、判断情報より大きくしすぎない。
- 控え役者は `次に来そう` だけを小さく表示し、通常の控え説明は抑える。

### 兆候3灯

- `高/中/低` 文字はアクセシビリティと詳細には残す。
- 表面では灯りの強さ、アイコンの明度、並び順で強度を表す。
- 準備カード側にも同じ3灯を小さく表示し、どの準備がどの兆候を覆うかをチェック/消灯で示す。

### 準備カード

カード表面の常時表示は以下に限定する。

```txt
アイコン + 準備名
兆候カバー 3灯
活きる対応アイコン
短い狙い
前回マーカー
```

例:

```txt
注視
● ● ●
本番: 拾うアイコン
熱を拾う
```

- `見えている兆候に合う` などの文章は、選択中状態またはメモ紙へ寄せる。
- `本番で拾う` は行動固定に見えやすいので、カード表面では `拾うアイコン + 活きる` 程度にする。
- 危険/別筋は赤文字説明ではなく、カード外枠とカバー灯の欠けで先に伝える。

### 本番前メモ

- 現行のメモ紙は残す。
- モバイルでは初期状態を少し短くし、選択中準備の詳細確認として扱う。
- 承認CTA押下時は、現行どおりメモ紙上の承認欄が `未承認 -> 承認済` へ変わる。
- 承認演出は短く、再演モードでは現行の短縮テンポを維持する。

## 対応画面仕様

対象: `ActorStage` のイベント表示、`ResponsePanel`, `ResponseChoiceCard`, `ResponseConsole`

### 発生イベント

- 対応画面の主役はイベント名にする。
- `本番で起きた`、イベント名、イベントアイコン、役者/状態の順に読む。
- イベント説明は3行まで。詳細は進行卓側に寄せる。
- 可能ならイベント種別ごとに光の動きや背景ラインを変える。

### 対応カード

カード表面の常時表示は以下に限定する。

```txt
対応アイコン + 対応名
勝ち筋ラベル
結果レンジレール
準備関係
主要影響2件
前回ゴースト
```

勝ち筋ラベル:

| 対応 | 表示 | 役割 |
| --- | --- | --- |
| 拾う | 熱を渡す | 評判を伸ばす攻め筋 |
| 整える | 呼吸を戻す | 流れ/負荷の安定筋 |
| 待つ | 間を残す | 信頼/余韻の伸び筋 |
| 切る | 次へ送る | 事故/負荷の収束筋 |

- `攻め手/安定手` のような抽象バッジは増やさない。
- 結果レンジレールは現行を維持し、現在見込みの位置をより強く光らせる。
- 主要影響は現行の `評/流/信/負` 短縮を維持する。
- 危険警告はカード下部に強く表示する。ただし通常時に事故文言を出しすぎない。

### 進行卓

- 選択中対応の詳細を読む場所として維持する。
- 相性盤、影響ゲージ、ビルド、ほころび、前回比は進行卓側に置く。
- モバイルでは進行卓の上部に、選択中対応の要約を1行で固定表示する。
- 進行卓内にCTAは入れない。現行どおり送出バーで分離する。

## 場面結果画面仕様

対象: `ResultPreviewCard`

### ファーストビュー構成

結果画面の上部は報酬画面として再構成する。

1. 結果ランク
2. 場面タイトル
3. 星/拍手/客席熱量
4. ルート: 役者 -> 出来事 -> 対応
5. 差分4つ
6. 次の判断メモ

理由、代償、申し送り、客席反応、準備回収詳細、公演の色、主な理由はその下に置く。

### 報酬演出

| 条件 | 表示 |
| --- | --- |
| 名場面 | 照明が広がる、星/拍手を強く出す |
| 場面化 | 舞台灯が安定する |
| 小さな成功 | 進行表チェックを出す |
| ほころび | 残った系統アイコンを次ターンへ渡す |
| 事故 | 暗転気味。ただし次の判断メモをすぐ見せる |
| 準備が活きた | メモ紙由来のスタンプを大きく出す |
| 新場面候補 | 図鑑スタンプを出す |
| ほころび回収 | ほころび系統アイコンを回収演出する |

### 差分表示

- 現行の `Delta` は維持する。
- 表面では `評判 +3` より先に `評判が伸びた` を読ませる。
- 負荷はプラスが悪化、マイナスが改善であることが一目で分かる色にする。

## 終演画面仕様

対象: `ResultScreen`, `NextPerformancePanel`, `PerformancePoster`

### ファーストビュー

終演画面の上部は次の順にする。

1. 公演タイトル
2. ランク/総合点/次ランクまで
3. 次の挑戦カード
4. 今回の公演バッジ
5. 主要指標

レビュー本文は残すが、モバイルでは初期表示を1文から2文に抑える。

### 次の挑戦カード

`NextPerformancePanel` をタイトル画面にも展開し、見た目を統一する。

表示内容:

```txt
次の挑戦
2日目ソワレを場面化
締め -> 整える / 総合+6
[同じ巡り合わせで詰める]
```

- 推薦ロジックは `nextChallengeRecommendation` を使う。
- タイトル画面のホームでも、`新しい公演を始める` と推薦CTAを横並びにせず、推薦カードを主導線として出す。
- 新規プレイヤーだけは `初日のマチネを開ける` を主目標にする。

### 指標

- `準備 5/6`, `名場面 1`, `場面以上 3`, `ほころび 1` は残す。
- ただしファーストビューでは `あと5点でS+` や `準備あと1回` のような課題文を指標より上に置く。
- 詳細の観客アンケート、劇評、判断傾向、タイムラインは現行どおり折りたたみに置く。

## タイトル画面仕様

対象: `TitleScreen`

### ホーム

- 最初に `次の挑戦カード` を表示する。
- 次に主CTAを1つ表示する。
- 新規開始、再演、日替わりを横並びの同格ボタンにしない。
- 記録、図鑑、遊び方は現行タブを維持する。

### 初回導線

- 初回は遊び方を長く読ませず、ホームのカードで次を示す。

```txt
最初の目標
若手の兆候を読んで、準備を1つ通す
[初日のマチネを開ける]
```

- 遊び方タブは現行の3ステップを維持し、必要な人だけ読める場所にする。

## コピー仕様

### カード表面で使う短い言葉

| 用途 | 文言例 |
| --- | --- |
| 準備の狙い | 熱を拾う / 間を残す / 乱れを戻す / 次へ渡す |
| 準備関係 | 活きる / 外でも効く / 合わない |
| 兆候カバー | 3/3 ではなく3灯で表示。ariaには `兆候3件中3件に備え` |
| 結果見込み | 名場面狙い / 場面化狙い / 小さく成功 / 崩れ抑え / 事故注意 |
| 影響 | 評+ / 流+ / 信+ / 負- |
| 次の挑戦 | 2日目ソワレを場面化 / 準備ヒット4回以上 / 最終負荷2以下 |

### 詳細層に残す言葉

- 役者タイプ説明
- 状態の意味
- 備えどおり/外れた場合の読み筋
- 相性盤4軸
- 決め手、代償、客席反応
- 公演レビュー、劇評、タイムライン

## アイコン/記号仕様

既存の `Icon` を優先して使う。追加する場合は同じ線幅、角丸、`currentColor` の方針に揃える。

追加候補:

| 対象 | 表現 |
| --- | --- |
| 状態 | 高揚/沈思/不安/没入/疲労の状態灯 |
| 公演の色 | 熱量/余韻/精度/収束の型紋章 |
| ほころび | 光/音/進行/道具の系統アイコン |
| 前回比 | 上昇/同じ/下降の小マーカー |
| 新規解放 | 図鑑スタンプ |

## モーション仕様

- モーションは判断を待たせない。
- 重要な瞬間だけ短く入れる。
- `prefers-reduced-motion` では点灯/色変化だけにする。

| 瞬間 | 推奨演出 |
| --- | --- |
| 準備選択 | 準備カードのカバー灯が点く |
| 準備承認 | メモ紙の承認欄に押印 |
| イベント発生 | イベントカードに照明スイープ |
| 対応選択 | 結果レンジの現在位置が点灯 |
| キュー送出 | 送出バー左右ランプが流れる |
| 名場面 | 結果タイトル周辺に短い照明拡張 |
| ほころび回収 | 系統アイコンが結果票へ吸い込まれる |
| ベスト更新 | 終演カードにスタンプ |

## 完了基準

- 440px幅のファーストビューで、準備画面は焦点役者、兆候3件、準備4択の上半分以上が見える。
- 360px幅でも主要CTAのテキストがはみ出さない。
- 対応画面で、進行卓を読まなくても4択の伸び筋/危険筋が分かる。
- 結果画面で、ランク、場面タイトル、差分、次の判断メモがスクロール前に見える。
- 終演画面で、ランク直後に次の挑戦カードが見える。
- タイトル画面で、履歴がある時は次の挑戦が主導線として見える。
- 詳細説明、劇評、タイムラインは残り、舞台裏の雰囲気が薄くならない。
- `npm run check:ui` が通る。
