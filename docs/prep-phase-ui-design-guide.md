# 準備フェーズ UI デザインガイド

この文書は、準備フェーズの UI を今後変更するときの判断基準を整理したデザインガイドです。

準備フェーズは、役者の兆候を見て、本番中の想定外へ備える準備を選び、その内容を本番前メモとして承認する場面です。単なる選択画面ではなく、「舞台裏スタッフが本番前に対応方針を整え、進行へ回す」体験として扱います。

## 目的

- 役者の兆候から、どの準備が本番中に活きそうかを読めるようにする。
- 準備と本番中の対応の関連を、行動固定ではなく「活きる関係」として伝える。
- 選んだ準備が本番前メモに反映され、承認されて本番へ進む流れを明確にする。
- 行動に対応フェーズとは似すぎないように、準備フェーズは「読む、記録する、承認する」体験として差別化する。

## 体験の流れ

1. 役者カードで、焦点役者、状態、役者信頼、見えている兆候を確認する。
2. 準備カードで、備える兆候、備える揺れの種類、活きやすい対応アイコンを見る。
3. 選んだ準備が本番前メモに反映される。
4. メモの承認欄は最初は未承認として表示する。
5. `この準備で本番へ` を押す。
6. 今表示されているメモ紙の承認欄に `承認済` が押され、本番中の対応フェーズへ進む。

## 画面構成

### 役者カード

役者カードは、次の順で読む設計にする。

1. 役割バッジ
2. 役者名
3. 状態ブロック
4. 役者信頼/パッシブ
5. 見えている兆候

`焦点役者` はカード左上のバッジとして表示する。右上に状態バッジを置くと情報が散るため避ける。

`控え` と `次に来そう` もバッジとして表示する。特に `次に来そう` は次ターンの注意対象なので、通常の説明文ではなく視認性のあるバッジにする。

状態は、役者タイプ説明とは分けて小ブロック化する。状態の意味説明は役者カード上に出しすぎず、本番前メモに置く。

```text
状態
高揚
拾うと伸びやすい
```

`勢い型。...` のような役者タイプ説明は、詳細層に寄せる。名前の直下は役者名、状態、役者信頼、兆候を優先する。

### 準備カード

準備カードは、以下の情報だけに絞る。

- 現場作業指示のタイトル
- 具体的な現場メモ
- 対象
- 作業
- 注意
- 備える兆候の灯り
- メモ反映中の状態

推奨表示:

```text
前明かりを残す準備
予定外の一歩や一言が出たら、照明で追えるよう卓と袖の合図を合わせておく。
対象 前進 / アドリブ
作業 照明待機 / 呼び込み保留
● ● ●
メモ反映中
```

カード表面では、`注視`、`余白`、`締め`、`転換` などの内部カテゴリ名を出さない。内部カテゴリ名は採点、ログ、詳細分析のための分類であり、プレイヤーが通常の選択時に覚える情報ではない。

`本番対応 拾う` や `本番で拾う` は、次の行動が決まっているように見えやすいため避ける。

`拾う備え`、`受け方は拾う` などの重複表現は避ける。初見で「何を受けるのか」が分かりにくく、カード内の情報量も増える。

`兆候との相性` は、見えている兆候と準備の噛み合いを見る軸として詳細層で使う。カード表面では灯りで示し、対象と作業は「前進」「照明待機」「呼び込み保留」のような現場語彙で示す。

### 本番前メモ

本番前メモは、バインダーに挟まれた生成りのメモ紙として表現する。

役割は、選択中の準備内容を確認し、決定事項として承認すること。単なる詳細パネルではない。

メモ内には以下を表示する。

- `本番前メモ`
- 現場作業指示のタイトル
- `見えている兆候`
- `同じ準備で拾える出来事`
- 準備の読み筋
- `備えどおりに来たら`
- `外れたら`
- `承認欄`

承認欄は最初は `未承認` とする。

```text
承認欄    未承認
```

CTA 押下後は、同じメモ紙上で `承認済` に変わる。

```text
承認欄    承認済
```

承認演出は全画面カードへ切り替えない。メモ紙とのつながりが切れ、承認欄に押された意図が伝わりにくくなるため。

### CTA

主 CTA はメモ外に置く。

```text
この準備で本番へ
```

メモ内の承認欄は状態表示であり、押せる UI として扱わない。押せる操作は主 CTA に集約する。

CTA 押下後は、周囲の準備カードを少し落とし、メモ紙を強調したまま承認欄へ `承認済` を押す。これにより、プレイヤーは「メモの内容を承認して本番へ進む」ことを理解しやすい。

## 文言ルール

### 推奨する文言

- `準備を決める`
- `役者の兆候を見て、本番中の想定外に備える準備を選ぶ。`
- `兆候との相性`
- `見えている兆候に合う`
- `一部に備えあり`
- `今の兆候とは遠い`
- `熱の揺れ`
- `間の揺れ`
- `進行の乱れ`
- `転換の崩れ`
- `メモ反映中`
- `本番前メモ`
- `同じ準備で拾える出来事`
- `備えどおりに来たら`
- `外れたら`
- `承認欄`
- `未承認`
- `承認済`
- `この準備で本番へ`

### 避ける文言

- `先読み`
- `一手目` / `二手目`
- `主対応`
- `袖の備え`
- `拾う備え` / `待つ備え`
- `受け方は拾う`
- `本番対応 拾う`
- `本番で拾う`
- `本番で拾うと活きる`
- `準備候補`
- `張っている出来事`
- `ほかに備える兆候`
- `別の揺れ`
- `備えた出来事なら:`
- `違う出来事なら:`

理由:

- `一手目` / `二手目` はフェーズタブと見出しで代替できるため冗長。
- `主対応` は副対応があるように見え、関係性が分かりにくい。
- `拾う備え` は、準備なのか本番対応なのかが曖昧。
- `本番で拾う` は行動が決まっているように見える。
- `別の揺れ` は用語として孤立しやすい。
- コロン書きはメモ感よりもシステム説明感が強くなる。

## 視覚ルール

### 色と質感

- 準備フェーズ全体は暗い舞台裏の上に、生成りのメモ紙を置く。
- メモ紙は明るくしてよいが、白すぎる紙は避ける。舞台裏の暖色照明になじませる。
- 承認スタンプは赤茶系のインクにする。金色の CTA と役割を分ける。

### カード密度

- 準備カードは比較しやすさを優先する。
- 本番前メモは詳細確認を優先する。
- 同じ情報をカードとメモで繰り返す場合は、カード側を要約、メモ側を詳細にする。

### 状態表示

- 役者状態はカード右上の孤立バッジにしない。
- 役者名周辺のメタ情報として扱うが、役者タイプ説明と混ざらないように独立ブロックにする。
- 状態には意味を添える。

例:

```text
状態
没入
拾う・待つが効きやすい
```

## インタラクション

### 準備カード選択

カードをタップすると、即確定ではなくメモに反映される。

選択中カードには `メモ反映中` を表示する。`選択中` よりも、メモとの関係が伝わる。

### 準備確定

`この準備で本番へ` を押すと、メモ紙の承認欄が `未承認` から `承認済` に変わる。

推奨演出:

- 準備カード側を少し暗くする。
- メモ紙を軽く強調する。
- 承認欄の `承認済` を短く押印アニメーションする。
- その後、行動に対応フェーズへ進む。

全画面の承認カードは使わない。承認欄との関係が切れてしまう。
