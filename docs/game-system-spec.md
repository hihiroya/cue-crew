# ゲームシステム仕様

この文書は、現在の実装から読み取れるゲームシステム仕様をまとめたものです。対象は `src/game` 配下のゲームロジック、ゲーム進行を制御する `src/app/App.tsx`、およびゲーム画面コンポーネントです。

## ゲーム概要

- タイトル: `本番中につき！ 〜舞台裏の一手〜`
- ジャンル: 1人用の舞台裏マネジメントゲーム
- プレイヤーの役割: 本番中の裏方。役者へ直接命令するのではなく、兆候を読み、本番前の準備を決め、予定外の出来事へ対応する。
- 1公演の長さ: 全3日、各日マチネ/ソワレの2公演、合計6ターン。
- 1ターンの基本構造:
  1. 焦点役者と兆候を確認する。
  2. 「準備」を1つ選び、本番前メモへ反映する。
  3. 役者の出来事が解決される。
  4. 本番中の出来事に対して「行動に対応」する。
  5. 場面結果を確認し、スコアへ反映する。
- 勝敗条件: 明示的なゲームオーバーはない。6ターン終了時の公演結果、レビュー、ハイライトによって評価される。

## 公演日構成

| 日 | 名前 | 役割 | ターン |
| --- | --- | --- | --- |
| 1 | 1日目 | 呼吸作り。準備と座組信頼が公演の型を作る | 1-2 |
| 2 | 2日目 | 揺らぎ。型を活かして攻めるか、負荷を整える | 3-4 |
| 3 | 3日目 | 千秋楽。残った信頼と負荷を回収して閉じる | 5-6 |

日と公演回は総ターンから計算する。

- `act = ceil(totalTurn / 2)`。内部名は既存互換のため `act` のままだが、UIでは「日」として扱う。
- `turnInAct = 1` はマチネ、`turnInAct = 2` はソワレ。

各日ソワレの結果確定後、裏方負荷が1軽減される。

| 公演回 | 役割 | 結果表示 |
| --- | --- | --- |
| マチネ | 昼公演。見極めと調整が効きやすい | 薄め。ソワレへ残る手応えを中心に表示する |
| ソワレ | 夜公演。評判は伸びるが、負荷も残りやすい | 厚め。その日の評判、公演の型への影響を表示する |

1日目ソワレの結果確定時に「公演の型」を決める。型は2日目以降の判断傾向を少し変える。

| 型 | 条件傾向 | 効果 |
| --- | --- | --- |
| 熱量で押す公演 | 拾う、評判が高い | 拾う成功で評判が伸びやすいが、負荷も残りやすい |
| 間で残す公演 | 待つ、座組信頼が高い | 待つ成功で座組信頼が伸び、準備が外れても崩れにくい |
| 段取りで支える公演 | 整える、段取りや負荷回復が高い | 整える成功で負荷が抜けやすく、ほころびを抑えやすい |
| 崩れを閉じる公演 | 切る、低負荷で閉じる | 切るで事故を抑えやすいが、評判は伸びにくい |

## 進行状態

ゲーム状態 `status` は以下の5種類。

| 状態 | 内容 |
| --- | --- |
| `title` | タイトル画面。新規開始、履歴から同じseedで再演、遊び方表示ができる。 |
| `prep` | 準備フェーズ。焦点役者、状態、発生しやすい兆候上位3件、役者の癖、次ターン候補、準備カード、本番前メモを表示する。 |
| `response` | 行動に対応フェーズ。準備確定後に役者イベントが確定し、4つの対応を選ぶ。 |
| `result` | 結果プレビュー表示。マチネはソワレへの影響を薄く、ソワレはその日の評判と型への影響を厚く表示する。 |
| `finished` | 終演画面。最終結果、レビュー、ハイライト、再演操作を表示する。 |

操作アクションは以下。

| アクション | 有効状態 | 効果 |
| --- | --- | --- |
| `START` | 任意 | 新しい公演を開始する。seed指定時はそのseedで開始する。 |
| `RESET_TO_TITLE` | 任意 | タイトル状態へ戻る。 |
| `SELECT_PREP` | `prep` | 準備を保存し、役者イベントを解決して `response` へ進む。UI上は本番前メモの承認欄が `未承認` から `承認済` になってから進む。 |
| `SELECT_RESPONSE` | `response` または `result` | 対応を保存し、`result` へ進む。 |
| `COMMIT_RESULT` | `result` | 結果を確定し、スコア・ログ・役者状態・次ターンを更新する。 |

## スコア

管理する公演スコアは4種類。

| スコア | 初期値 | 範囲 | 意味 |
| --- | --- | --- | --- |
| 評判 `sceneScore` | 0 | 制限なし | 場面化、名場面化の積み上げ。ターン結果では「場面」として表示する。 |
| 段取り `flowScore` | 0 | 制限なし | 公演全体の進行の安定度。ターン結果では「流れ」として表示する。 |
| 座組信頼 `trustScore` | 0 | 制限なし | 役者との信頼、終盤の伸びやすさ。ターン結果では「信頼」として表示する。 |
| 裏方負荷 `backstageLoad` | 0 | 0-5 | 裏方の負担。高いほど結果判定に不利。 |

裏方負荷は `clampLoad` により常に0以上5以下に丸められる。

## 役者

役者は3人。

| ID | 表示名 | 初期状態 | 初期信頼 | 初期疲労 | 初期役割 |
| --- | --- | --- | --- | --- | --- |
| `lead` | 主役 | 沈思 | 0 | 0 | 焦点 |
| `junior` | 若手 | 高揚 | 0 | 0 | 兆候 |
| `skilled` | 技巧派 | 没入 | 0 | 0 | 待機 |

役者が持つ値。

| 値 | 内容 |
| --- | --- |
| `state` | 現在の心理・身体状態。 |
| `trust` | 個別役者の信頼値。内部的に蓄積されるが、現時点では直接スコア計算に使われない。 |
| `fatigue` | 0-3の疲労値。3以上になると状態が疲労へ固定される。 |
| `currentRole` | 画面上の役割。`focus`、`sign`、`waiting`。 |

### 役者状態

| ID | 表示名 |
| --- | --- |
| `elated` | 高揚 |
| `contemplative` | 沈思 |
| `anxious` | 不安 |
| `immersed` | 没入 |
| `fatigued` | 疲労 |

状態変化候補は `高揚 -> 沈思 -> 不安 -> 没入 -> 疲労` の配列からランダムに選ばれる。循環順に進むのではなく、この配列を候補リストとして使う。

## 焦点役者

各ターンの焦点役者はseed付き乱数で決まる。優先順は以下。

| ターン | 優先焦点 |
| --- | --- |
| 1 | 若手 |
| 2 | 主役 |
| 3 | 技巧派 |
| 4 | 若手 |
| 5 | 主役 |
| 6 | 技巧派 |
| 7 | 若手 |
| 8 | 技巧派 |
| 9 | 主役 |
| 10 | 若手 |

判定手順。

1. `seed:focus:totalTurn` から乱数を作る。
2. 72%の確率で優先焦点を採用する。
3. 28%の確率で上記の優先順リスト全体からランダムに選ぶ。

焦点以外の役者の表示役割は以下。

- 焦点役者は `focus`。
- 焦点以外で、若手または状態が不安の役者は `sign`。
- それ以外は `waiting`。

## 役者イベント

イベントは8種類。

| ID | 表示名 | 説明 |
| --- | --- | --- |
| `stepForward` | 前へ出る | 予定より半歩前に出て、客席の視線をさらいかけている。 |
| `adlib` | アドリブ | 台詞の隙間に、台本にはない一言が落ちた。 |
| `heatUp` | 熱が乗る | 言葉と身体に熱が宿り、場面が大きく膨らみ始めた。 |
| `silence` | 沈黙する | 台詞が止まり、舞台に息を呑むような間が生まれた。 |
| `positionShift` | 立ち位置がズレる | 立ち位置が少しズレ、照明と視線の軸が変わった。 |
| `tempoRush` | テンポが走る | 呼吸より先に台詞が走り、舞台全体の拍が揺れている。 |
| `delayedExit` | 退場が遅れる | 退場の一歩が遅れ、余韻と進行の境目で揺れている。 |
| `ensembleWaver` | 群像が揺れる | 周囲の動きがわずかに乱れ、群像の輪郭が揺らいだ。 |

イベント解決は以下。

1. 焦点役者の種類別重みと状態別重みを足す。
2. 焦点役者の疲労が2より大きい場合、`positionShift`、`tempoRush`、`ensembleWaver` にそれぞれ+3する。
3. `seed:event:totalTurn:focusId` から乱数を作る。
4. 合計重みによる加重抽選を行う。

準備は役者イベントの抽選には影響しない。準備は「起きた出来事に備えられていたか」と「本番中の対応へどうつながるか」の判定に使う。

## 準備

準備フェーズでは4種類から1つ選ぶ。準備はイベント抽選seedには含まれず、結果スコアの準備品質判定に影響する。

| ID | 表示名 | 本番で活きる対応 | 外れた時の受け方 | 備える出来事 |
| --- | --- | --- | --- | --- |
| `watch` | 注視 | 本番で拾うと活きる | 整えるで安全寄りに処理できる | 前へ出る、アドリブ、熱が乗る |
| `makeSpace` | 余白 | 本番で待つと活きる | 熱が来たら拾うと伸びる | 沈黙する、退場が遅れる、熱が乗る |
| `tightenFlow` | 締め | 本番で整えると活きる | 高負荷なら切って守る手もある | 立ち位置がズレる、テンポが走る、群像が揺れる |
| `prepareTransition` | 転換 | 本番で切ると活きる | 低負荷なら整える・待つで場面化を狙える | テンポが走る、退場が遅れる、群像が揺れる、立ち位置がズレる |

発生イベントが選択した準備の対応イベントに含まれる場合、結果スコアに+2される。結果画面では `準備が活きた`、`準備が一部活きた`、`別の備えだった` といった関係を表示する。

準備カードには、準備名、本番で活きる対応、兆候との相性、メモ反映状態を表示する。対応表示は「次の行動が固定された」という意味ではなく、「この準備は本番でその対応を選ぶと活かしやすい」という関係を示す。

選択中の準備は本番前メモへ反映される。メモには見えている兆候、同じ準備で拾える出来事、備えどおりに来た場合、外れた場合、承認欄を表示する。CTA `この準備で本番へ` を押すと、メモ紙上の承認欄が `未承認` から `承認済` へ変わり、行動に対応フェーズへ進む。

## 行動に対応

準備確定後、役者イベントに対して4種類から1つ選ぶ。

| ID | 表示名 | 説明 | 主な負荷系統 |
| --- | --- | --- | --- |
| `catch` | 拾う | 予定外の行動を見せ場に変える | 光 |
| `arrange` | 整える | 乱れを舞台全体の呼吸に戻す | 進行 |
| `wait` | 待つ | 間や余韻を信じて急かさない | 音 |
| `cut` | 切る | 場面を閉じ、崩れを次へ送る | 進行 |

対応は役者イベントとの相性、役者タイプ、役者状態、公演日/公演回、公演の型、連続使用、信頼、負荷、ほころび回収に影響する。

対応カードには、各候補について以下を表示する。

- 手の性格。
- 選んだ準備との関係。
- その対応で狙うもの。
- 成功幅。
- 上振れ・下振れ。
- 出来事との相性。
- 役者・状態との相性。
- 公演日/公演回の影響。
- 負荷・信頼・流れへの副作用。

画面上では、対応カードは候補比較、進行卓 HUD は選択中の対応を送出する判断面として分ける。進行卓 HUD には、選択中の対応、準備とのつながり、送出見込み、判定ランプとしての相性盤、送出ゲージとしての送出後の影響、CTA `この対応を送る` を表示する。

相性盤は今後の高度な判断に必要な情報として扱うため、折りたたまない。`出来事`、`役者型`、`状態`、`公演回` の4項目を常時表示し、判定ランプ、`◎ 強い`、`○ 合う`、`△ 普通`、`× 注意` のような記号と短い言葉を併記する。

### イベント相性

| イベント | 拾う | 整える | 待つ | 切る |
| --- | ---: | ---: | ---: | ---: |
| 前へ出る | 3 | 2 | 0 | 0 |
| アドリブ | 3 | 0 | 2 | -2 |
| 熱が乗る | 3 | 2 | 0 | 0 |
| 沈黙する | 0 | 2 | 3 | 0 |
| 立ち位置がズレる | 2 | 3 | -2 | 2 |
| テンポが走る | 2 | 3 | -2 | 2 |
| 退場が遅れる | 2 | 0 | 3 | 2 |
| 群像が揺れる | 0 | 3 | -2 | 2 |

### 画面上の判断理由

対応カードには、各候補の性格、成功幅、判断理由が出る。最終結果ランクはここでは出さず、結果画面で初めて表示する。

| 項目 | 表示例 |
| --- | --- |
| 手の性格 | `攻め手` |
| 準備との関係 | `準備が活きる`、`別筋で受ける`、`準備とは遠い` |
| 狙い | `予定外を見せ場に変える` |
| 成功幅 | `ほころび〜小さな成功` |
| 上振れ | `噛み合えば場面化まで届く` |
| 下振れ | `負荷が残る` |
| 相性盤 | `出来事 ◎ 強い`、`役者型 ○ 合う`、`状態 ○ 合う`、`公演回 △ 普通` |
| 副作用 | `負荷-1 / 信頼+1` |

候補カードのトーンは成功幅と危険警告に応じて変わる。`事故` は通常の見込みとしては表示せず、下振れが事故圏内の時だけ `危険: 事故圏内` として表示する。

| 状態 | トーン |
| --- | --- |
| 名場面まで伸びる | best |
| 場面化まで届く | good |
| ほころびが絡む | thin |
| 事故圏内警告 | danger |

## 結果スコア計算

結果プレビュー時、内部スコア `score` を以下の合計で計算する。

```txt
score =
  イベント相性
  + 役者タイプ補正
  + 役者状態補正
  + 準備品質補正
  + 公演日/公演回補正
  + 公演の型補正
  + 信頼補正
  + 負荷ペナルティ
  + 連続使用ペナルティ
  + ほころび回収補正
```

### 役者タイプ補正

| 条件 | 補正 |
| --- | ---: |
| 主役に `wait`、`catch`、`arrange` | +1 |
| 若手に `catch`、`arrange` | +1 |
| 技巧派に `wait`、`arrange` | +1 |
| 高揚状態の若手に `wait` | -1 |

### 役者状態補正

| 条件 | 補正 |
| --- | ---: |
| 高揚に `catch` | +1 |
| 沈思に `wait` | +1 |
| 不安に `arrange` | +1 |
| 没入に `wait` または `catch` | +1 |
| 疲労に `arrange` または `cut` | +1 |
| 疲労に `catch` | -1 |

### 準備品質補正

準備は3段階で評価する。

| 品質 | 条件 | 効果 |
| --- | --- | --- |
| 活きた | 発生イベントが選択した準備の対応イベントに含まれる | +2。上振れ幅が広がり、名場面まで届く。 |
| 一部活きた | 発生イベントとは違ったが、見えている兆候には備えていた | 補正0。場面化まで安定しやすい。 |
| 別の備えだった | 発生イベントにも見えている兆候にも備えていなかった | -1。上限が小さな成功まで下がり、攻め手は負荷が残りやすい。 |

一部活きた場合の上限は場面化、別の備えだった場合の上限は小さな成功。準備が活きた時は上限なし。

### 準備と対応の関係

各準備には、本番で選ぶと活きやすい対応がある。UIでは「本番で拾うと活きる」のように表示し、行動固定ではなく相性として示す。内部的には外れた場合の受け方も残す。

| 準備 | 活きやすい対応 | 効き方 | 外れた時の受け方 |
| --- | --- | --- | --- |
| 注視 | 拾う | 準備が活きた時に場面の伸びを少し広げる | 整えるで安全寄りに処理できる |
| 余白 | 待つ | 信頼と余韻を守りやすい | 熱が乗る出来事なら拾う攻め筋もある |
| 締め | 整える | 乱れを流れへ戻しやすい | 高負荷時は切る守り筋もある |
| 転換 | 切る | 下振れ、ほころび、負荷悪化を抑えやすい | 低負荷なら整える・待つで場面化を狙える |

準備が活きたうえで対応も噛み合った場合、`prepareTransition + cut` 以外は追加で+1する。`prepareTransition + cut` は点数を伸ばす手ではなく、崩れを閉じる手として扱うため、スコア加点ではなく下振れ軽減を持つ。

### 公演日/公演回補正

| 条件 | 補正 |
| --- | ---: |
| 1日目で `wait` または `arrange` | +1 |
| 1日目ソワレで `catch` | +1 |
| 2日目で `catch` | +1。ソワレならさらに強い |
| 2日目で `arrange` または `cut` | +1 |
| 3日目で `wait` または `arrange`、かつ信頼スコアが2以上 | +2 |
| 3日目で `cut`、かつ負荷3以上 | +1 |

### 公演の型補正

1日目ソワレ後に決まった型は、2日目以降の対応に小さな補正を与える。

| 型 | スコア補正 | 結果差分 |
| --- | --- | --- |
| 熱量で押す公演 | `catch` +1 | `catch` 成功時、評判+1、負荷+1 |
| 間で残す公演 | `wait` +1 | `wait` 成功時、座組信頼+1 |
| 段取りで支える公演 | `arrange` +1 | `arrange` 成功時、負荷-1追加 |
| 崩れを閉じる公演 | `cut` +1 | `cut` 時、負荷悪化とほころびを抑えやすい |

### 信頼補正

```txt
trustBonus = clamp(floor(trustScore / 3), -2, 2)
```

信頼スコアが高いほど最大+2、低いほど最大-2まで結果スコアに影響する。

### 負荷ペナルティ

| 裏方負荷 | 補正 |
| ---: | ---: |
| 0-2 | 0 |
| 3 | -1 |
| 4 | -2 |
| 5 | -3 |

### 連続使用ペナルティ

現在選んだ対応を含めて、同じ対応が何回連続しているかを `count` とする。2連続までは戦術として許容し、3連続以上で内部スコアに-1する。

| 対応 | 2連続 | 3連続以上 |
| --- | --- | --- |
| `catch` | スコア補正なし | -1 |
| `arrange` | スコア補正なし | -1 |
| `wait` | スコア補正なし | -1 |
| `cut` | スコア補正なし | -1 |

直前と違う対応を選んだ場合、`count = 1` となりペナルティはない。

### ほころび回収補正

`pendingFrayEvent` があり、対応が `catch`、`wait`、`arrange` のいずれかなら+1。

## 結果ランク

内部スコアから結果ランクを決める。

| 内部スコア | ランクID | 表示名 | 星 |
| ---: | --- | --- | --- |
| 7以上 | `masterpiece` | 名場面 | ★★★★★ |
| 4-6 | `scene` | 場面化 | ★★★★☆ |
| 2-3 | `smallSuccess` | 小さな成功 | ★★★☆☆ |
| 0-1 | `fray` | ほころび | ★★☆☆☆ |
| -1以下 | `accident` | 事故 | ★☆☆☆☆ |

## 結果差分

結果ランクごとの基本差分。

| ランク | 場面 | 流れ | 信頼 |
| --- | ---: | ---: | ---: |
| 名場面 | +4 | +2 | +2 |
| 場面化 | +3 | +1 | +1 |
| 小さな成功 | +1 | +1 | 0 |
| ほころび | 0 | -1 | -1 |
| 事故 | -1 | -2 | -2 |

結果画面では差分を数字だけで出さず、以下の3点を併記する。

- 体感ラベル: `見せ場級`、`呼吸が戻った`、`重い代償` など。
- 4段階メーター: `4/4` のように評価幅内の位置を見せる。
- 内部数値: 対戦・攻略で必要になるため `+4`、`-1` の数値も残す。

場面の `+4` は `最大` ではなく `見せ場級` と表現する。これは計算上の最大幅は示しつつ、演出的には「この場面が客席に残った」ニュアンスを優先するため。

### 負荷差分

| 対応 | 成功系ランク | 低結果系ランク |
| --- | ---: | ---: |
| `catch` | 名場面なら+2、それ以外は+1 | +2 |
| `arrange` | -1 | +1 |
| `wait` | -1 | +1 |
| `cut` | 基本±0 | 基本±0 |

成功系ランクは `masterpiece`、`scene`、`smallSuccess`。低結果系ランクは `fray`、`accident`。

追加で以下がかかる。

| 条件 | 効果 |
| --- | --- |
| 2日目で `catch` | 負荷+1。揺らぎの日では攻めの代償が大きい。 |
| 3日目で `arrange` または `cut` が成功 | 負荷-1。千秋楽では立て直しが効く。 |
| マチネで `arrange` または `wait` が成功 | 負荷-1追加。昼公演は調整が効きやすい。 |
| ソワレで `catch` が成功 | 負荷+1追加。夜公演は評判が伸びるぶん負荷が残る。 |
| 別の備えだった状態で `catch` | 負荷+1。無理に拾うと負荷が残る。 |
| `prepareTransition` で `cut` | 負荷増加がある場合、+1分を抑制する。 |

`wait` 成功時は信頼+1、`cut` 成功時は信頼-1。負荷上昇は低結果だけでなく、攻めの代償として扱う。

### 連続使用による結果差分補正

同じ対応を直前ターンから続けた場合、以下を結果差分へ追加する。

| 対応 | 2連続 | 3連続以上 |
| --- | --- | --- |
| `catch` | 追加負荷 +1 | 追加負荷 +2、流れ -1 |
| `arrange` | 成功時の負荷 -1 を無効化 | 負荷 +1、場面 -1 |
| `wait` | 成功時の負荷 -1 を無効化 | 負荷 +1、流れ -1 |
| `cut` | 信頼 -1 | 信頼 -1、負荷 +1 |

対応カードの副作用欄にも、`連続使用: 負荷+1`、`連続使用: 負荷回復なし`、`連続使用: 信頼-1` のように事前表示する。結果画面では連続使用補正が発生した理由を内訳に表示する。

### 追加ペナルティ

| 条件 | 効果 |
| --- | --- |
| 結果確定前の裏方負荷が4以上、かつランクが名場面ではない | 流れ差分 -1 |

### 転換 + 切るの下振れ軽減

`prepareTransition` を選び、かつ対応で `cut` を選んだ場合、場面スコアの上振れは控えめにしつつ、下振れを抑える。

| 条件 | 効果 |
| --- | --- |
| `accident` 相当、かつ準備が活きたまたは一部活きた | `fray` へ軽減する |
| `fray` 相当、かつ準備が活きた | `smallSuccess` へ軽減する |
| ほころび生成判定 | 通常より発生率を下げる |
| 負荷増加 | +1分を抑制する |

この組み合わせは名場面狙いではなく、崩れを小さく閉じて次へ送る判断として扱う。

## ほころび

ほころびは高負荷時に次ターン以降へ持ち越される未解決イベント。

### 生成条件

結果確定後の負荷見込み `nextLoad = clamp(backstageLoad + deltaLoad)` を使う。

1. `nextLoad` が4未満なら生成しない。
2. 結果ランクが名場面なら生成しない。
3. 結果ランクが `fray` または `accident` なら必ず生成する。
4. それ以外のランクでは48%の確率で生成する。

ほころびの系統は `preview.loadBias`、なければ現在の `state.loadBias`、それもなければ `stageManagement` を使う。

### ほころび系統とタイトル

| 系統 | 表示名 | 候補タイトル |
| --- | --- | --- |
| `light` | 光 | 照明が一歩遅れた / スポットの輪郭が揺れた |
| `sound` | 音 | 音が感情を先取りしすぎた / 残響が少し長く残った |
| `stageManagement` | 進行 | 転換が急ぎすぎた / 進行の合図が一拍ずれた |
| `props` | 道具 | 小道具の受け渡しが遅れた / 小道具の準備が詰まった |

現実装では対応から直接発生する負荷系統は `light`、`sound`、`stageManagement` の3種類。`props` は型とほころびタイトルとして定義されているが、通常の対応からは直接選ばれない。

### 回収

ほころびが残っている状態で、対応が `catch`、`wait`、`arrange` なら結果スコアに+1される。結果ランクが事故以外なら、場面タイトル生成時にほころび系統を「回収された系統」として使う。

## 役者状態更新

結果確定後、次ターンに入る前に役者状態を更新する。乱数seedは `seed:state:totalTurn:logs.length`。

各役者について以下を行う。

### 疲労更新

- 26%の確率で疲労-1。
- 疲労は0-3に丸める。
- 裏方負荷は役者の疲労を直接増やさない。負荷は舞台裏側のリスクとして扱い、役者ステータスとは切り離す。

### 個別信頼更新

- 焦点役者で、公演全体の信頼スコアが0より大きい場合、個別信頼+1。
- それ以外では個別信頼は変化しない。

### 状態更新

- 焦点役者は72%の確率で状態を変更する。
- 焦点以外は34%の確率で状態を変更する。
- 状態変更時は5状態からランダムに選ぶ。
- 疲労が3以上なら、選ばれた状態に関係なく `fatigued` になる。

## 場面タイトル

場面タイトルは専用テンプレートのうち、条件に合う最初のものを使う。条件は、役者、イベント、対応、結果ランク、回収したほころび系統の任意組み合わせ。

主な専用タイトル例。

| 条件 | タイトル |

| 若手 + 前へ出る + 拾う | 勢いが生んだ見せ場 |
| 若手 + アドリブ + 拾う | 拾われたアドリブ |
| 主役 + 沈黙する + 待つ | 観客が息を呑む沈黙 |
| 主役 + 退場が遅れる + 待つ | 余韻を残す退場 |
| 技巧派 + 立ち位置がズレる + 整える | 意味を持った立ち位置 |
| 技巧派 + 群像が揺れる + 整える | 揺らぎを包む群像 |
| テンポが走る + 切る | 鮮やかな暗転 |
| 沈黙する + 待つ | 客席まで届いた間 |
| 光のほころび回収 + 名場面 | 遅れて差した光 |
| 音のほころび回収 + 場面化 | 沈黙に寄り添う音 |
| 進行のほころび回収 + 名場面 | 転換が生んだ余白 |

| ランク | 汎用タイトル |

タイトル用乱数seedは `seed:title:totalTurn:score`。

## 終演結果

6ターン目、つまり3日目ソワレの結果確定後、状態は `finished` になる。終演結果には以下を含む。

| 項目 | 内容 |
| --- | --- |
| `seed` | 公演seed |
| `finishedAt` | ISO文字列の終了日時 |
| `sceneScore` | 最終場面スコア |
| `flowScore` | 最終流れスコア |
| `trustScore` | 最終信頼スコア |
| `backstageLoad` | 最終裏方負荷 |
| `performanceStyle` | 初日ソワレ後に決まった公演の型 |
| `title` | 公演タイトル |
| `review` | 公演レビュー |
| `highlights` | 最大5件のハイライト |

### 公演タイトル

以下の優先順で決まる。

| 条件 | タイトル |
| --- | --- |
| 名場面が2回以上、かつ `catch` が2回以上 | 予定外を評判に変えた公演 |
| `wait` が2回以上、かつ信頼スコアが4以上 | 余韻が残る千秋楽 |
| `catch` が3回以上 | 熱量で押し切った公演 |
| `arrange` が3回以上、かつ流れスコアが場面スコア以上 | 段取りが支えた公演 |
| 流れスコアが0未満、または信頼スコアが0未満 | 荒れたが忘れがたい三日間 |
| 上記以外 | 三日間を支えきった公演 |

### 公演レビュー

レビューは3つの文を連結して作る。

1. `catch` 回数が `wait` 回数以上なら、予定外を拾う判断が熱を与えた旨。そうでなければ、待つ判断が余韻を残した旨。
2. `arrange` が3回以上なら、乱れを整えて流れを守った旨。そうでなければ、揺れが本番らしさになった旨。
3. `masterpiece` または `scene` のログがあれば、終盤側から最大3件の場面タイトルを挙げる。なければ、派手な名場面は少ないが舞台は途切れなかった旨。

### ハイライト

ハイライトは以下の手順で最大5件選ぶ。

1. 全ログを結果ランク順に並べる。ランク順は `masterpiece` > `scene` > `smallSuccess` > `fray` > `accident`。
2. 同ランクでは場面差分が大きいものを優先する。
3. 上位5件を取る。
4. 表示順は総ターン昇順に戻す。

## 履歴保存

終演時、公演結果は `localStorage` に保存される。

- キー: `honban.performance.history.v1`
- 保存件数: 最新8件まで。
- 保存形式: `PerformanceResult[]` のJSON。
- 読み込み失敗時は空配列として扱う。

タイトル画面では履歴を表示し、履歴項目を押すと同じseedで再演できる。

## seedと乱数

乱数は文字列seedから決定的に生成される。

- seed生成: `honban-${Date.now().toString(36)}-${random}`。
- 乱数実装: `xmur3` による文字列ハッシュと、`mulberry32` 系の数値更新。
- 同じseedで開始すると、同じ選択をした限り同じ焦点役者、イベント、タイトル、状態変化が再現される。

用途ごとにseed文字列を分けている。

| 用途 | seed形式 |
| --- | --- |
| 焦点役者 | `seed:focus:totalTurn` |
| 役者イベント | `seed:event:totalTurn:focusId` |
| 場面タイトル | `seed:title:totalTurn:score` |
| ほころび | `seed:fray:totalTurn:nextLoad` |
| 役者状態更新 | `seed:state:totalTurn:logs.length` |

## 画面仕様

### タイトル画面

表示内容。

- `はじめる` ボタン。
- `遊び方` ボタン。押すたびに説明パネルを開閉する。

### ゲーム画面

常時表示。

- ヘッダー: タイトル、現在の日、マチネ/ソワレ、公演回の役割、公演の型、`公演を降りる` ボタン。

- ほころびが残っている場合、`前のほころび: タイトル` を表示する。

`prep` では焦点役者カードを表示する。

- 役割バッジ。焦点役者はカード左上に表示する。
- 役者名。
- 役者タイプの説明。
- 状態ブロック。`状態`、状態名、状態の意味を表示する。

兆候上位3件はイベント重みを降順に並べる。表示強度は以下。

- 最上位の重みとの差が2以内なら `高`。
- 差が8以内なら `中`。
- それ以外は `低`。

`response` では焦点役者カードに確定イベントを表示し、対応選択カードを表示する。

`result` では役者カードを非表示にし、結果プレビューカードを表示する。
